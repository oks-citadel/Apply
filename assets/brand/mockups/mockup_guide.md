## Mockup Best Practices

### 1. Use Realistic Content

‚úì **Do**:
- Actual company logos (with permission)
- Real job titles and descriptions
- Realistic user names and photos
- Accurate statistics
- Proper formatting

‚úó **Don't**:
- Lorem ipsum placeholder text
- "John Doe" generic names
- Fake company names
- Unrealistic numbers (e.g., "999,999 applications")

### 2. Consistent Styling

‚úì **Do**:
- Use brand colors throughout
- Consistent typography
- Proper spacing/padding
- Aligned elements
- Cohesive design system

‚úó **Don't**:
- Mix different UI styles
- Inconsistent button styles
- Varying color schemes
- Misaligned elements

### 3. Show Real User Flows

‚úì **Do**:
- Demonstrate actual features
- Show before/after states
- Include micro-interactions
- Display loading states
- Error states when relevant

‚úó **Don't**:
- Only show perfect scenarios
- Hide important UI elements
- Fake functionality
- Overly simplified interfaces

### 4. Context Matters

‚úì **Do**:
- Show UI in realistic environments
- Include device context
- Add subtle shadows/reflections
- Use appropriate backgrounds
- Consider viewing angle

‚úó **Don't**:
- Floating screenshots with no context
- Harsh shadows or effects
- Busy backgrounds competing with UI
- Unrealistic perspectives

### 5. Accessibility Visible

‚úì **Do**:
- Show good contrast ratios
- Include focus states
- Display keyboard navigation
- Demonstrate screen reader compatibility
- Show text sizing options

## Screenshot Capture Process

### 1. Preparation
- Clear browser cache
- Use incognito/private mode (clean state)
- Set viewport to exact size
- Disable browser extensions (except necessary)
- Use consistent browser (Chrome recommended)

### 2. Staging Environment
- Use staging/demo environment
- Populate with realistic data
- Ensure all features functional
- Test across devices

### 3. Capture Tools

#### Browser-Based
- Chrome DevTools (Device Mode)
- Firefox Responsive Design Mode
- Safari Responsive Design Mode

#### Browser Extensions
- Awesome Screenshot
- Fireshot
- GoFullPage (scrolling screenshots)

#### Desktop Apps
- Snagit
- CleanShot X (Mac)
- Greenshot (Windows)
- Xnapper (Mac)

#### Online Services
- BrowserStack (cross-browser)
- LambdaTest (device testing)
- Sizzy (responsive testing)

### 4. Post-Processing

#### Image Editing
- Crop to exact dimensions
- Add device frames (optional)
- Subtle shadows for depth
- Compress for web use
- Maintain readable text

#### Annotation (if needed)
- Arrows/callouts for features
- Numbered steps
- Highlight key elements
- Add descriptive text overlay

## Mockup Export Specifications

### Web Use (Marketing, Blog, Social)
- **Format**: PNG with transparency or JPG
- **Resolution**: @2x for retina (1920px ‚Üí export at 3840px)
- **Compression**: TinyPNG or similar
- **File size**: < 500KB per image
- **Color profile**: sRGB

### Print Use (Brochures, Pitch Decks)
- **Format**: PNG or TIFF
- **Resolution**: 300 DPI minimum
- **Color profile**: CMYK (for print) or sRGB (for PDF)
- **File size**: High quality (compression minimal)

### Presentations (PowerPoint, Keynote)
- **Format**: PNG
- **Resolution**: 1920√ó1080 or 2560√ó1440
- **Compression**: Moderate (balance size/quality)
- **Background**: Transparent when possible

### Documentation (Help Center, Guides)
- **Format**: PNG
- **Resolution**: Actual size (no upscaling)
- **Compression**: Medium (readable text priority)
- **File size**: < 300KB per image
- **Alt text**: Include descriptive alt text
