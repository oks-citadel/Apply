# ApplyforUs Color System

## Overview
The ApplyforUs color system is designed to be vibrant, professional, and accessible. Our palette supports light and dark modes, conveys meaning through color, and maintains WCAG 2.1 AA accessibility standards throughout.

## Color Combinations

### Recommended Pairings

#### Primary Combinations
✓ **Indigo-500 + Violet-500**: Brand gradient, hero elements
✓ **Indigo-600 + Emerald-500**: CTA buttons with success indicators
✓ **Indigo-500 + Gray-800**: Primary action + text
✓ **Gray-50 + Indigo-500**: Light background + primary accent

#### Success Combinations
✓ **Emerald-500 + Emerald-50**: Success message
✓ **Emerald-600 + White**: Success button
✓ **Emerald-100 + Emerald-700**: Success banner

#### Warning Combinations
✓ **Amber-500 + Amber-50**: Warning message
✓ **Amber-600 + White**: Warning button
✓ **Amber-100 + Amber-800**: Warning banner

#### Error Combinations
✓ **Red-500 + Red-50**: Error message
✓ **Red-600 + White**: Destructive button
✓ **Red-100 + Red-800**: Error banner

#### Neutral Combinations
✓ **Gray-800 + White**: Primary text on light
✓ **Gray-50 + Gray-900**: Light background + text
✓ **Gray-100 + Gray-700**: Card + text
✓ **White + Gray-800**: Dark text on white

### Avoid These Pairings
✗ **Red + Green**: Accessibility issues (colorblind)
✗ **Low contrast grays**: Below 4.5:1 ratio
✗ **Pure black on pure white**: Too harsh (prefer Gray-800)
✗ **Multiple bright colors**: Overwhelming, no hierarchy

## Gradients

### Brand Gradients

#### Primary Gradient (Indigo to Violet)
```css
background: linear-gradient(135deg, #6366F1 0%, #8B5CF6 100%);
```
**Usage**: Hero sections, primary CTAs, feature cards

#### Success Gradient (Emerald to Teal)
```css
background: linear-gradient(135deg, #10B981 0%, #14B8A6 100%);
```
**Usage**: Success states, achievement badges, growth indicators

#### Warm Gradient (Amber to Orange)
```css
background: linear-gradient(135deg, #F59E0B 0%, #F97316 100%);
```
**Usage**: Highlights, energy elements (use sparingly)

#### Subtle Background Gradient
```css
background: linear-gradient(180deg, #F9FAFB 0%, #FFFFFF 100%);
```
**Usage**: Page backgrounds, subtle depth

#### Dark Background Gradient
```css
background: linear-gradient(180deg, #1F2937 0%, #111827 100%);
```
**Usage**: Dark mode backgrounds, headers

### Gradient Best Practices
- Default angle: 135° (diagonal) for energy, 180° (vertical) for subtle
- Avoid more than 2-3 colors in a gradient
- Maintain readability of text over gradients
- Test on both light and dark backgrounds
- Use overlays for text readability

## Accessibility Guidelines

### Contrast Ratios (WCAG 2.1)

#### Level AA (Minimum)
- **Normal text** (< 18px regular, < 14px bold): 4.5:1
- **Large text** (≥ 18px regular, ≥ 14px bold): 3:1
- **UI components and graphics**: 3:1

#### Level AAA (Enhanced)
- **Normal text**: 7:1
- **Large text**: 4.5:1

### ApplyforUs Standard
We aim for **WCAG 2.1 Level AA** minimum, with AAA where feasible.

### Color Blindness Considerations

#### Types of Color Blindness
- **Protanopia** (red-blind): 1% of males
- **Deuteranopia** (green-blind): 1% of males
- **Tritanopia** (blue-blind): Rare
- **Achromatopsia** (total color blindness): Very rare

#### Our Approach
- Never rely on color alone to convey information
- Use icons + color for status indicators
- Provide text labels alongside colored elements
- Test designs with colorblind simulators
- Use patterns or textures in addition to color in charts

#### Safe Color Combinations
✓ Indigo + Emerald (blue + green friendly)
✓ Amber + Indigo (distinguishable)
✓ Red + Blue (high contrast)

## Dark Mode

### Dark Mode Philosophy
Dark mode reduces eye strain in low-light environments and can save battery on OLED screens. Our dark mode is carefully crafted for readability and brand consistency.

### Dark Mode Palette

#### Backgrounds
- **Primary**: Gray-900 (#111827)
- **Secondary**: Gray-800 (#1F2937)
- **Elevated**: Gray-700 (#374151)
- **Pure black**: #000000 (OLED optimization, optional)

#### Text
- **Primary**: White (#FFFFFF) or Gray-50 (#F9FAFB)
- **Secondary**: Gray-400 (#9CA3AF)
- **Tertiary**: Gray-500 (#6B7280)
- **Disabled**: Gray-600 (#4B5563)

#### Brand Colors (Adjusted for Dark)
- **Primary**: Indigo-400 (#818CF8) instead of Indigo-500
- **Secondary**: Violet-400 (#A78BFA) instead of Violet-500
- **Success**: Emerald-400 (#34D399) instead of Emerald-500
- **Warning**: Amber-400 (#FBBF24) instead of Amber-500
- **Error**: Red-400 (#F87171) instead of Red-500

### Dark Mode Implementation

```css
/* Light mode (default) */
:root {
  --color-bg-primary: #FFFFFF;
  --color-bg-secondary: #F9FAFB;
  --color-text-primary: #1F2937;
  --color-text-secondary: #6B7280;
  --color-brand-primary: #6366F1;
}

/* Dark mode */
@media (prefers-color-scheme: dark) {
  :root {
    --color-bg-primary: #111827;
    --color-bg-secondary: #1F2937;
    --color-text-primary: #F9FAFB;
    --color-text-secondary: #9CA3AF;
    --color-brand-primary: #818CF8;
  }
}

/* Manual toggle */
.dark {
  --color-bg-primary: #111827;
  --color-bg-secondary: #1F2937;
  --color-text-primary: #F9FAFB;
  --color-text-secondary: #9CA3AF;
  --color-brand-primary: #818CF8;
}
```

## Usage Guidelines

### Do's

✓ **Use semantic color naming**
```css
--color-success: #10B981;
--color-error: #EF4444;
--color-warning: #F59E0B;
```

✓ **Maintain contrast ratios**
- Always check text contrast
- Use tools like WebAIM Contrast Checker

✓ **Use CSS variables**
- Easier theme switching
- Centralized color management

✓ **Test on actual devices**
- Colors appear different on various screens
- Test in light and dark environments

✓ **Provide color alternatives**
- Icons with text labels
- Patterns in addition to color
- Multiple indicators of state

### Don'ts

✗ **Don't use color alone to convey meaning**
- Add icons, text, or patterns

✗ **Don't use too many colors**
- Stick to palette
- Avoid random colors

✗ **Don't ignore accessibility**
- Low contrast is not acceptable
- Test with colorblind simulators

✗ **Don't hardcode color values**
- Use design tokens/variables
- Easier to maintain and theme

✗ **Don't use pure black on white** (in body text)
- Too harsh, prefer Gray-800 on White
- Better reading experience
