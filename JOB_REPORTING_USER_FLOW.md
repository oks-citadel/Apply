## Key Interactions

### 1. Prevention of Duplicate Reports
- Backend sets `isReported: true` on Job after first report
- Frontend displays "Reported" button (disabled)
- Button cannot be clicked again

### 2. Real-time Updates
- React Query automatically refetches data
- Admin sees updated status immediately
- No manual refresh needed

### 3. Validation & Error Handling
- Form validates before submission (Zod)
- API validates on backend
- User sees clear error messages
- Loading states during submission

### 4. Admin Authorization
- Admin routes protected by middleware
- Non-admin users cannot access /admin/job-reports
- API endpoints check admin role
